multi-purpose (mostly games) Discord Bot

I created this Discord bot as a project to learn Python and "using" the Discord API (never actually learned it because I just used the discord.py module to interface with the API).

Originally, this was going to be a general purpose tool that can automatically assign roles, start polls, send funny memes, etc. However, as I ran out of ideas, I just began thinking of games I wanted to play and implementing them into the bot so that I can play all of them in one platform (Discord).

General information about the LeoBot:

  • Written in Python
  • Uses discord.py module to interface with Discord API
    • Means the functionalities are separated into "cogs" which can be loaded and unloaded from the bot
  • Self-hosted on a Raspberry Pi 3 Model A+
  • Bot data stored on USB flash drive attached to Pi
  • Stores all servers' data together in one JSON file (very not scalable idea but I'm not releasing the bot to the public so it's fine)

Functionalities (Cogs) of LeoBot:

  • General:
    • Assigns default role when user joins server
    • Sends message when user leaves server
  • Hangman: allows users on the server to play a game of hangman
  • Help: Sends message message detailing available commands (taken from a Gist by nonchris on Github)
  • Ligma: Responds to "daddy chill" messages with gifs
  • Markov:
    • Tries to "learn" how every user talks/sends messages
    • Creates a Markov chain for each user based on the messages that that user sends in the server
    • Basis of chain is the frequency a word is proceeded by another word in the messages that the individual sends (so the more you send messages, the more the bot "learns" you)
    • Can generate a sentence that is meant to "replicate" the way a specified user talks in the server using that user's Markov Chain
    • One of the functionalities I'm more proud of due to it's complexity and non meme-yness
  • Mordle (Match Wordle): A multiplayer implementation of Wordle
  • Poll: Allows users to create polls in their server
  • Utility: Allows users to set a default role for their server
  • Word Finder: A multiplayer implementation of Boggle (similar to Word Hunt on Game Pigeon on iPhones)
  • Wordle: Just Wordle
  • WTP (Who's that Pokémon?): A Pokémon guessing game where anyone can guess the Pokémon based on their silhouette

playing rock paper scissors on an MSP430

Out of all the projects on this website, this is the one project that actually requires some semblance of electrical engineering competence (albeit not much). As a general overview, this project involves an MSP430 microcontroller attached to 240 x 320 TFT display and five push buttons. Using the buttons, you can play "Rock, Paper, Scissors" against a random number generator.

For now, the extent of the features of this project go as far as keep tracking of the player's longest winning streak; however, the expandability potential of this project is vast. Just to name a few, I plan to (in no particular order):

  • Add more games to play
  • 3D print a casing for the entire console
  • Find a way to create a way to develop game files on a computer and transfer those files to and from the device to eliminate the need to store all the games on the device's memory/storage
  • Figure out a way to power the device using a battery pack and find a way to recharge the battery
  • Attach the touch pins of the dispaly to the microcontroller in order to enable the ability to develop touch based games

For now, the code is not publicly available because it's a hacky mess of spaghetti.

purposefully preposterous programming (p)language

YAYAYA is an original programming language based around the ever popular Brainfuck (and other similar esolangs). The intent behind the language was to make it fully pronounceable with the syllable "YA" (although most programs will sound the exact same when enunciated).

The attached GitHub link also includes a compiler I wrote which parses any specified file and displays the output of the program.

Like Brainfuck, there are eight different commands that manipulate data pointers and an instruction pointer. The biggest difference is that YAYAYA contains a delimiter (YAH) which separates each command. Also, all white space is ignored in the parsing of the source code.

Command Function
YA Increment data pointer
A Decrement data pointer
YAYA Increment byte at the data pointer
AYA Decrement byte at the data pointer
YAYAYA Output the byte at the data pointer
AYAYA Take one byte of input from stdin and store it at the data pointer
?? If the byte at the data pointer is 0, jump to the matching !!
!! If the byte at the data pointer is not 0, jump back to the first command after the matching ??
YAH Delimiter of commands
